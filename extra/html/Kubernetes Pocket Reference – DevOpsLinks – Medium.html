Kubernetes Pocket Reference

Alexey Nizhegolenko
Mar 12

v 0.2

Very often when you start using some new tech or product, you will learn a new commands and methods for doing right. You may spend some time and found non standard or especially yours list of needed commands and methods. But as time go on, it may be complicated to keep it all in mind, so you start to write it down. In time this notes grow up it to some kind of pocket reference.

In this small reference I combined a list of popular console commands for managing Kubernetes cluster. I started write it like my own list of useful commands, but maybe it‚Äôll be helpful for someone too, so I have convert it in some kind of article.

Also in first edition of this manual, it‚Äôll be only list of useful commands, but I have plans to add there a interesting workarounds or tricks examples in the future.

All this commands need to be run from configured kubectl utility.

0. About kubectl

Useful thing is to configure autocomplete, example for BASH:

# apt-get install bash-completion
# source <(kubectl completion bash)
# echo "source <(kubectl completion bash)" >> ~/.bashrc

Get full list of available commands with short description:

kubectl --help

Get information for the specific command:

kubectl command --help

Display or change kubeconfig settings:

kubectl config view

1. Cluster and system information.

Get version information, ‚ÄúServer Version‚Äù in output, will contains the apiserver version, which mean your Kubernetes server version:

kubectl version  --short

Get cluster relevant information:

kubectl cluster-info
# Detailed view (prepare for the huge output)
kubectl cluster-info dump

Get supported API versions in this cluster:

kubectl api-versions

Get supported API resources in this cluster:

kubectl api-resources
#More detailed info
kubectl api-resources -o wide
# API resources from all namespaces
kubectl api-resources --namespace=true

List existing nodes in cluster:

kubectl get nodes -o wide
#Short view of nodes list
kubectl get nodes
#Watch nodes continuously
kubectl get nodes -w
# Get all worker nodes only
kubectl get node --selector='!node-role.kubernetes.io/master'

Get last cluster events:

kubectl get events --sort-by=.metadata.creationTimestamp

Get detailed information about nodes:

kubectl describe node node-name
#Detailed information for all nodes in cluster
kubectl describe nodes

Get list of available namespaces:

kubectl get namespaces

Get detailed information about namespaces:

kubectl describe namespace namespace-name
#Information about all namespaces
kubectl describe namespaces

List all secrets in cluster:

kubectl get secrets --all-namespaces

Get the documentation for pods or svc manifests:

kubectl explain pods
kubectl explain svc

Show node metrics:

kubectl top node node-name
# For all nodes
kubectl top node

2. Check running resources in cluster.

Get list of existing Pods (default namespace):

kubectl get pods

Get list of existing Pods (specific namespace):

kubectl get pods -n namespace-name

Get list of all pods from all namespaces:

kubectl get pods --all-namespaces

Get list of pods with labels info:

kubectl get pods --show-labels

Get pods placed by nodes:

kubectl get pods -o wide ‚Äî sort-by=‚Äù{.spec.nodeName}‚Äù

Get pods sorted by some way, like restart count for example:

kubectl get pods --sort-by='.status.containerStatuses[0].restartCount'

Show detailed information about Pod:

kubectl describe pod pod-name -n namespace-name

Print the logs for a container in a pod:

kubectl logs pod-name -n namespace-name

Get list of the replication controllers:

kubectl get rc 
# list all replication controllers
kubectl get rc --all-namespaces
# in specific namespace
kubectl get rc -n namespace-name

Show details about replication controller:

kubectl describe rc rc-name

Get list of existing deployments:

kubectl get deployments
# List all deployments from cluster
kubectl get deployments --all-namespaces

Show details about deployment:

kubectl describe deployments deployment-name
# Same if deployment not in default namespace
kubectl describe deployments deployment-name -n namespace-name

Get list of services:

kubectl get svc
# More info 
kubectl get svc -o wide

Describe service:

kubectl describe svc service-name

3. Interacting with Pods.

Create a new pod, quick run from console:

# This command will create a new deployment with selected name also
kubectl run name --image=image-name
# New version of this command are
kubectl create deployment name --image=image-name

Expose just created pod port:

# This will create a new service
kubectl expose deployment name --port=external --target-port=internal

Scale created deployment:

kubectl scale --replicas=count deployment name

Create new pod from file:

kubectl create -f file-name.yaml

Run a commands on Pod:

kubectl exec pod-name some-command
# Run commands in interactive mode
kubectl exec -ti pod-name -- /bin/bash
# Run a command for the specific container in Pod(if Pod have more than one container)
kubectl exec pod-name some-command -c container-name

Attach to the running pod:

kubectl attach pod-name -i

Show metrics for a specific pod and its containers (if Heapster installed):

kubectl top pod pod-name --containers

4. Creating Kubernetes objects.

Create new service, deployment or other resources from manifest file:

kubectl create -f file.yaml
# From multiple files
kubectl create -f ./file1.yaml -f ./file2.yaml
# From a link
kubectl create -f http://some-url/file.yaml
# Create a deployment from command line
kubectl create deployment name --image=image-name

5. Updating/Editing/Deleting Kubernetes objects.

Updating the image in all containers of some deployment:

kubectl set image deployment/name containers-name=image:v2

Rollback to the previous deployment type:

kubectl rollout undo deployment/name
# Watch the updating process
kubectl rollout status -w deployment/name

Force delete and then re-create the resource:

kubectl replace --force -f ./some-pod.json

Update a single-container pod‚Äôs image version:

kubectl get pod pod-name -o yaml | sed 's/\(image: new-image\):.*$/\1:v4/' | kubectl replace -f -

Add or change Pod label:

kubectl label pods pod-name new-label=label-name

Scale deployment once:

kubectl scale deployment name ‚Äî replicas=2
# Scale multiple resources
kubectl scale --replicas=5 deployment/name1 deployment/name2

Set autoscale for some deployment:

kubectl autoscale deployment name --min=2 --max=10

Also You can edit any API resource in an editor:

# Edit some service for example
kubectl edit svc/some-svc-name

Delete any resource:

kubectl delete some-resource

Delete deployment and service with same name, also will stop and delete all pods:

kubectl delete deployment,svc name

Delete a single pod, same for the svc or other resource:

kubectl delete pod name

Delete pods, services with specific label

kubectl delete pods,services -l name=mylabe
# Same but with uninitialized also
kubectl delete pods,services -l name=mylabe --include-uninitialized

Delete all pods/services from the specific namespace

kubectl -n namespace-name delete po,svc --all

6. Interacting with Nodes

Drain some node in cluster for maintenance, respecting the application-level disruptions:

kubectl drain node-name

Mark node as unschedulable:

kubectl cordon node-name

Mark node schedulable back, after some maintenance:

kubectl uncordon node-name

Delete node from cluster:

kubectl delete node name

Add new node to cluster:

kubeadm join --token some-token master-ip:master-port

Add some taint to a node, this marks that the node should not accept any pods that do not tolerate the taints:

kubectl taint nodes node-name some-key:value

Allow mater nodes to run pods:

kubectl taint nodes --all node-role.kubernetes.io/master-
